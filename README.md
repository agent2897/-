# Репозиторій лабораторних робіт

Цей репозиторій містить файли лабораторних робіт з дисципліни професійної підготовки.

## Структура

- `Лабораторна робота 1/` — документи з лабораторної роботи 1
- `Лабораторна робота 2/` — документи з лабораторної роботи 2
- `Лабораторна робота 3/` — документи з лабораторної роботи 3, зокрема UML-діаграми класів

## Аналіз патернів (Лабораторна робота №3)

У діаграмі класів видно такі підходи:

- Абстрактні класи `Person` і `Staff` використовуються для центрального опису загальних атрибутів та операцій, що дозволяє зменшити повторення коду та забезпечити поліморфізм.
- Наслідування `Detainee`, `DutyOfficer` та `MedicalStaff` підтримує розширення поведінки для специфічних ролей системи.
- `INotificationService` як інтерфейс забезпечує відокремлення логіки оповіщень від конкретної реалізації.
- `Repository<T>` представляє шаблонний репозиторій для зберігання та доступу до сутностей, що відповідає шаблону Repository/DAO.
- Композиції `DetentionRecord` → `Event` і `DetentionRecord` → `Property` гарантують, що ці об’єкти належать лише конкретному запису утримання.
- Агрегація `DetentionRecord` → `Document` вказує, що документи можуть існувати незалежно від конкретного запису.

## Як надати доступ викладачам

1. Створити репозиторій на GitHub або GitLab.
2. Додати цей локальний репозиторій як `origin`.
3. Виконати `git push -u origin main`.
4. Надати викладачам доступ до репозиторію за допомогою GitHub/GitLab.
